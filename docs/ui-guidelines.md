# UI 规范

## 设计方向

系统视觉应当克制、清晰、可扫描，适合企业用户长时间处理列表、审批、统计和配置任务。首屏直接进入工作台或业务页面，不使用营销落地页结构。

## 布局

- 顶部栏高度：56px。
- 侧边栏宽度：248px，折叠宽度：72px。
- 内容区最大宽度不强行收窄，数据页面使用全宽布局。
- 页面主间距：24px；紧凑区块间距：16px；表单项间距遵循 Ant Design 默认密度。
- 卡片圆角不超过 8px。
- 不使用卡片嵌套卡片；页面区块使用普通容器或全宽区域承载。

## 色彩

- 基础语义色：主色 `#2563eb`，成功 `#16a34a`，警告 `#d97706`，危险 `#dc2626`，信息辅助 `#0891b2`。
- 文字和边界：文本主色 `#172033`，文本次级 `#5c667a`，边框 `#d8dee8`。
- 背景和表面：页面背景 `#f5f7fb`，页面表面 `#ffffff`。
- 浅底色：主色浅底 `#e8f0ff`，成功浅底 `#eefbf2`，警告浅底 `#fffbeb`，危险浅底 `#fef2f2`，中性浅底 `#f7f9fc`。
- 动作色：链接动作使用主色，正向业务动作使用成功色，工具动作使用中性色，危险动作使用危险色。

颜色使用保持功能化：状态颜色只用于状态本身，不大面积铺陈。

## 按钮颜色

- 查询、提交、保存等当前区域主动作使用主色 `#2563eb`。
- 新建、添加等正向业务动作可使用成功色 `#16a34a`，但优先采用浅底描边样式，避免与查询主按钮竞争。
- 导入、导出、刷新、批量处理等工具动作使用中性浅底、边框色和次级文本色。
- 危险动作只使用危险色 `#dc2626`，并优先放在确认弹窗或二次确认中。
- 不新增页面局部色值；如确实需要新颜色，先进入全局色板再使用。
- 表格行内操作统一使用全局链接动作色；`更多` 与普通行内操作同色，只通过箭头表达下拉属性，不使用灰色弱化。

## 字体

- 字体族：系统默认 sans-serif。
- 页面标题：20px / 28px，字重 600。
- 区块标题：16px / 24px，字重 600。
- 正文：14px / 22px。
- 辅助文本：12px / 18px。
- 字间距保持 0。

## 组件使用

- 主要命令使用 Ant Design Button，必要时搭配 lucide 图标。
- 图标按钮必须有 tooltip 或明确的可访问标签。
- 筛选条件使用 Form + Select/Input/DatePicker，不把筛选项做成装饰性卡片。
- 表格默认提供分页、状态列、操作列和空状态。
- 复杂详情使用 Drawer；危险或中断性操作使用 Modal。
- 二元选项使用 Switch 或 Checkbox；多选集合使用 Checkbox.Group 或 Select multiple。
- 枚举状态使用 Tag，颜色按业务语义而不是随机分配。

## 数据页面标准结构

1. 页面标题与主操作。
2. 指标或概览条。
3. 筛选区域。
4. 数据表格或主要工作区。
5. 抽屉或弹窗承载详情、编辑、审批。

更细的页面类型标准见 [页面标准](page-standards.md)。

## 响应式

- 桌面优先，保证 1280px 宽度下高效工作。
- 小屏下侧边栏转为窄栏，表格保留横向滚动。
- 工具栏允许换行，但按钮文本不得溢出容器。
