# 评分表配置

## 基本信息

| 项 | 内容 |
| --- | --- |
| 所属系统 | OA |
| 一级目录 | OA |
| 二级目录 | 配置管理 |
| 菜单路由 | `/oa/config/scorecards` |
| 文档状态 | 持续迭代 |
| 最近更新 | 2026-05-04 |

## 当前需求

- 支持统一维护多类考核评分表模板。
- 支持配置基础字段、自定义字段、评分人、考核维度、考核事项、评分标准和分值。
- 支持草稿中、启用中、已停用三类模板状态。
- 支持模板发布前校验和评分表预览。

## 字段与操作

| 类型 | 内容 |
| --- | --- |
| 列表字段 | 模板编码、评分表名称、状态、负责人、字段数、评分人、总分、版本、更新时间、说明 |
| 顶部操作 | 新建评分表、查询、重置 |
| 行内操作 | 编辑、预览、复制、发布、停用、删除 |
| 编辑操作 | 保存配置、预览填写窗口；编辑表单不维护状态、编码和版本 |

## 业务规则

- 新建模板默认为草稿中。
- 模板状态由系统动作自动维护：新建为草稿中，发布后为启用中，停用后为已停用；编辑表单不提供状态字段。
- 评分表配置页不绑定具体业务类型，各业务按自身场景选择已发布模板编码发起评分。
- 评分表编码由系统统一生成，生成时机为模板发布；草稿状态不生成编码，列表中以灰色“模板发布后生成”提示。
- 评分表编码规则为 `SCORE-YYYYMMDD-序号`，例如 `SCORE-20260505-001`。
- 每次编辑保存自动更新模板版本号，编辑表单不允许手工维护版本。
- 通过阈值、优秀阈值和结果名称不在通用模板中配置，由使用评分表的业务自行定义。
- 一张评分表至少保留一个评分人，最多支持五个评分人。
- 每张评分表至少包含一个考核维度，每个维度至少包含一个考核事项，每个事项至少包含一个评分标准。
- 分值只配置在评分标准层级，分值必须为非负数字。
- 模板基础信息、字段、评分人和考核内容完整后才允许发布启用。
- 已停用模板不再用于新建评分，但仍保留查询和归档。
- 删除模板需二次确认，且仅未发布的草稿模板允许删除。
- 系统字段支持搜索、勾选和排序，以适配字段数量较多的业务场景；系统字段均由系统值自动带出，不区分必填和选填。
- 负责人从用户列表中下拉选择。
- 各业务模块通过通用评分填写窗口打开已配置模板，窗口按模板动态渲染基础字段、评分人、考核维度、考核事项和评分标准。
- 评分填写窗口中的系统字段自动带出且不可编辑，预览标签仅展示字段名；自定义字段可填写。
- 评分明细按每条评分标准逐行填写，考核维度、考核事项采用类似 Excel 合并单元格的汇总展示。
- 多个评分人不分块展示，评分列按评分人数量横向展开，表头按“评分人（权重xx%）”命名；未配置权重时展示“评分人（等权重）”。
- 配置人可在编辑抽屉中直接预览填写窗口，也可在列表中使用“预览”验证业务评分时的实际填写效果。
