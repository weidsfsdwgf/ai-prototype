## 1. 项目背景

低值易耗品覆盖办公文具、会议物资、清洁耗材、后勤备品等高频低单价物品。当前这类物品容易分散在人工台账、即时沟通和线下登记中，存在库存不准、领用追踪困难、借用归还不闭环、损耗无法复盘、安全库存预警不及时等问题。

建设低值易耗品管理能力，目标是将物品分类、入库、申领、分发、借用、归还、损耗和安全库存统一沉淀为可查询、可追溯、可运营的线上台账，为行政和财务提供稳定的资产消耗管理基础。

## 2. 项目目标

- 核心目标 1：建立低值易耗品全生命周期台账，覆盖新增入库、补库、申领、分发、归还、损耗和库存状态。
- 核心目标 2：提升行政人员处理库存和申请的效率，减少线下登记、人工核算和重复确认。
- 核心目标 3：通过安全库存和流水记录提升物资补库、损耗追踪和区域管理的可见性。

### 成功指标

| 指标 | 目标值 | 说明 |
| --- | --- | --- |

| 关键操作留痕率 | 100% | 入库、分发、归还、损耗均生成对应记录 |

| UAT 阻塞缺陷 | 0 个 | 上线前不得存在 P0/P1 阻塞缺陷 |

## 3. 用户画像

### 用户群体：行政管理人员

- **特征**：负责物品采购入库、库存维护、申领分发、借用归还和损耗登记。
- **痛点**：物品类型多、区域分散、借用归还容易遗漏，人工表格难以及时反映真实库存。
- **使用场景**：每日查看待分发、低库存物品，处理员工申领，月底核对库存和损耗。

### 用户群体：部门员工

- **特征**：按工作需要申请办公、会议或后勤物品，关注申请是否被分发。
- **痛点**：不知道物品库存和处理状态，借用物品归还过程缺少提醒和记录。
- **使用场景**：发起领用或借用申请，说明用途和数量，等待行政分发。

### 用户群体：财务/资产负责人

- **特征**：关注资产消耗、库存异常、补库依据和损耗原因。
- **痛点**：难以从分散记录中快速判断损耗是否合理、库存是否超储或短缺。
- **使用场景**：查看物品流水、区域库存、损耗记录和安全库存状态，支持盘点和成本分析。

## 4. 功能列表

### 4.1 核心功能（P0）

| 功能名称 | 功能描述 | 优先级 | 备注 |
| --- | --- | --- | --- |
| 物品台账查询 | 支持按编号、名称、区域、库存状态查询低值易耗品，并按分类树筛选列表 | P0 | 库存状态包含正常、低于下限、高于上限、待分发、待归还 |
| 分类维护 | 支持分类搜索、展开折叠、新增一级分类、新增下级分类、编辑分类和删除分类 | P0 | 用于维护办公用品、会议物资、后勤物资等层级 |
| 入库管理 | 支持已有物品补库和新增物品入库，新增物品自动生成 `XE-` 编号 | P0 | 入库后更新库存、累计入库和入库流水 |
| 申领管理 | 支持发起领用或借用申请，记录申请人、部门、数量和说明 | P0 | 申领先进入待分发，不直接扣减库存 |
| 分发管理 | 行政人员确认分发申请，领用进入已领用流水，借用进入待归还记录 | P0 | 分发数量不得超过申请数量和当前库存 |
| 归还管理 | 支持借用物品确认归还，登记归还数量、损耗数量和说明 | P0 | 归还数量回补库存，损耗数量进入损耗 |
| 损耗管理 | 支持自然损耗出库和归还损耗记录 | P0 | 损耗出库扣减库存并累计损耗 |
| 安全库存管理 | 支持设置安全库存上下限，并自动展示库存状态 | P0 | 上限必须大于等于下限 |

## 5. 用户故事

### 5.1 物品台账查询

```gherkin
Feature: 物品台账查询
  As a 行政管理人员
  I want 按条件快速定位低值易耗品
  So that 我可以及时处理库存、申领和异常状态

  Scenario: 按编号和区域查询物品
    Given 系统中存在多个区域的低值易耗品
    When 我输入物品编号并选择区域后点击查询
    Then 列表仅展示符合编号和区域条件的物品
    And 列表展示库存、库存状态、待分发、待归还等字段

  Scenario: 重置查询条件
    Given 我已设置编号、名称、区域和库存状态筛选
    When 我点击重置
    Then 查询条件被清空
    And 列表恢复为当前分类范围下的全部物品
```

### 5.2 分类维护

```gherkin
Feature: 分类维护
  As a 行政管理人员
  I want 维护低值易耗品分类树
  So that 物品可以按业务类别归档和查询

  Scenario: 新增下级分类
    Given 左侧分类树中存在办公用品分类
    When 我在办公用品下新增办公文具分类
    Then 分类树展示新增的下级分类
    And 后续新增或编辑物品时可以选择该分类

  Scenario: 删除分类
    Given 分类树中存在一个待删除分类
    When 我确认删除该分类
    Then 分类树不再展示该分类
    And 该分类的下级分类也不再展示
```

### 5.3 入库管理

```gherkin
Feature: 入库管理
  As a 行政管理人员
  I want 登记已有物品补库或新增物品入库
  So that 系统库存与实际入库保持一致

  Scenario: 已有物品补库
    Given 某物品当前库存为 8
    When 我选择该物品并登记入库数量 20
    Then 该物品库存更新为 28
    And 累计入库增加 20
    And 系统生成一条入库流水

  Scenario: 新增物品入库
    Given 当前最大物品编号为 XE-006
    When 我填写区域、分类、名称、规格、单位、数量、管理员和安全库存后确认入库
    Then 系统新增一条物品台账
    And 新物品编号自动生成为 XE-007
    And 初始库存等于本次入库数量
```

### 5.4 申领与分发管理

```gherkin
Feature: 申领与分发管理
  As a 行政管理人员
  I want 处理员工的领用和借用申请
  So that 物品发放后库存和流水可以自动更新

  Scenario: 提交领用申请
    Given 某物品库存充足
    When 员工填写申领人、部门、申请类型、数量和说明后提交
    Then 系统生成一条待分发申请
    And 物品待分发数量增加
    And 当前库存不立即扣减

  Scenario: 确认领用分发
    Given 某物品存在待分发的领用申请
    When 行政人员输入本次分发数量并确认分发
    Then 系统扣减物品库存
    And 累计已领用增加
    And 系统生成已领用流水

  Scenario: 分发数量超过库存
    Given 某物品当前库存为 5
    When 行政人员输入本次分发数量 6 并确认分发
    Then 系统阻止提交
    And 提示本次分发数量不能大于当前库存
```

### 5.5 归还管理

```gherkin
Feature: 归还管理
  As a 行政管理人员
  I want 登记借用物品的归还和损耗
  So that 借用闭环和库存数据保持准确

  Scenario: 正常归还借用物品
    Given 某借用记录未归还数量为 5
    When 行政人员登记归还数量 5 且损耗数量 0
    Then 物品库存增加 5
    And 待归还数量减少 5

  Scenario: 部分归还并登记损耗
    Given 某借用记录未归还数量为 5
    When 行政人员登记归还数量 3 且损耗数量 2
    Then 物品库存增加 3
    And 累计损耗增加 2
    And 系统生成归还损耗记录
```

### 5.6 损耗与安全库存管理

```gherkin
Feature: 损耗与安全库存管理
  As a 财务/资产负责人
  I want 查看和维护损耗与安全库存
  So that 我可以识别异常消耗和补库风险

  Scenario: 登记自然损耗
    Given 某物品当前库存为 18
    When 行政人员登记自然损耗数量 4 并填写说明
    Then 物品库存更新为 14
    And 累计损耗增加 4
    And 系统生成自然损耗记录

  Scenario: 设置安全库存
    Given 某物品当前安全库存下限为 10 上限为 60
    When 行政人员将下限设置为 12 且上限设置为 50
    Then 系统保存新的安全库存范围
    And 库存状态按新范围重新计算

  Scenario: 安全库存上限小于下限
    Given 我正在设置安全库存
    When 我输入下限 20 且上限 10
    Then 系统阻止保存
    And 提示上限必须大于等于下限
```

## 6. 验收标准

### 6.1 物品台账查询验收标准

#### 功能验收

- [ ] 支持按物品编号、物品名称、区域、库存状态组合查询。
- [ ] 支持通过分类树筛选当前分类及其下级分类物品。
- [ ] 查询结果必须展示区域、编号、名称、分类、规格、单位、库存、库存状态、安全库存、入库、已领用、损耗、待分发、待归还、行政管理人员和操作。

#### 非功能验收

- [ ] 1000 条以内台账数据查询响应时间不超过 1 秒。
- [ ] 列表横向字段较多时必须支持横向滚动，字段不重叠。

#### 测试验收

- [ ] 覆盖单条件、多条件、空条件和无结果查询场景。
- [ ] UAT 查询类用例通过率 100%。

### 6.2 分类维护验收标准

#### 功能验收

- [ ] 支持分类搜索、展开折叠、新增一级分类、新增下级分类、编辑分类和删除分类。
- [ ] 搜索分类时保留命中的分类路径。
- [ ] 删除分类后，该分类及其子分类从分类树移除。

#### 非功能验收

- [ ] 分类操作完成后页面无需刷新即可更新展示。
- [ ] 分类名称为空时不得保存。

#### 测试验收

- [ ] 覆盖新增、编辑、删除、搜索和空名称校验。

### 6.3 入库管理验收标准

#### 功能验收

- [ ] 已有物品入库后，库存和累计入库按入库数量增加。
- [ ] 新增物品入库后，系统自动生成 `XE-` 编号并建立首条入库流水。
- [ ] 入库数量必须为大于 0 的整数。
- [ ] 新增物品必须填写区域、分类、名称、规格、单位、数量、行政管理人员、安全库存和入库说明。

#### 非功能验收

- [ ] 入库弹窗字段在桌面宽度下双列展示，移动或窄屏下不出现内容重叠。
- [ ] 编号生成不得与已有物品编号重复。

#### 测试验收

- [ ] 覆盖已有物品补库、新增物品入库、数量非法和必填缺失场景。

### 6.4 申领与分发管理验收标准

#### 功能验收

- [ ] 申领支持领用和借用两种类型。
- [ ] 申领提交后生成待分发申请，增加待分发数量，不立即扣减库存。
- [ ] 确认领用分发后扣减库存、增加累计已领用并生成已领用流水。
- [ ] 确认借用分发后扣减库存、生成待归还记录并增加待归还数量。
- [ ] 分发数量不得大于申请数量或当前库存。

#### 非功能验收

- [ ] 分发操作必须有明确的数量校验提示。
- [ ] 待分发明细必须可追溯申请单号、申请人、部门、数量和申请时间。

#### 测试验收

- [ ] 覆盖领用申请、借用申请、领用分发、借用分发、超申请数量和超库存场景。

### 6.5 归还管理验收标准

#### 功能验收

- [ ] 借用分发后必须进入待归还记录。
- [ ] 确认归还时支持登记归还数量、损耗数量和说明。
- [ ] 归还数量增加库存，损耗数量不增加库存但增加累计损耗。
- [ ] 归还数量加损耗数量不得超过未归还数量。

#### 非功能验收

- [ ] 归还弹窗必须展示物品、申请单、申请人、部门和未归还数量。
- [ ] 无损耗归还也必须填写归还说明或默认说明。

#### 测试验收

- [ ] 覆盖全量归还、部分归还、归还损耗、超未归还数量和数量为负场景。

### 6.6 损耗与安全库存管理验收标准

#### 功能验收

- [ ] 自然损耗出库后扣减库存、增加累计损耗并生成损耗记录。
- [ ] 归还损耗与自然损耗统一进入损耗记录，并区分损耗类型。
- [ ] 安全库存支持设置下限和上限。
- [ ] 库存小于下限时状态为低于下限，大于上限时状态为高于上限，否则为正常。
- [ ] 安全库存上限不得小于下限。

#### 非功能验收

- [ ] 库存状态标签颜色应清晰区分正常、低于下限和高于上限。
- [ ] 损耗记录必须保留单号、类型、数量、说明、操作人和操作时间。

#### 测试验收

- [ ] 覆盖自然损耗、归还损耗、安全库存修改、低于下限、高于上限和非法上下限场景。

## 7. 附录

### 7.1 术语表

| 术语 | 说明 |
| --- | --- |
| 低值易耗品 | 单价较低、使用频繁、消耗周期短或管理粒度较轻的办公、会议、后勤物品 |
| 入库 | 将采购或补充物品登记到系统库存 |
| 领用 | 员工领取后不需要归还的使用方式 |
| 借用 | 员工领取后需要归还或登记损耗的使用方式 |
| 待分发 | 申领已提交但行政尚未确认发放的数量 |
| 待归还 | 借用已分发但尚未全部归还或损耗闭环的数量 |
| 自然损耗 | 因存储、运输、过期、破损等原因导致的库存损耗 |
| 归还损耗 | 借用物品归还时登记的损坏或缺失数量 |
| 安全库存 | 用于识别库存过低或过高的上下限范围 |
