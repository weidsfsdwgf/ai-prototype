## 当前需求

- 支持维护绩效指标库，供评分表配置选择引用。
- 支持按指标分类树查询指标，指标分类可新增、编辑和删除。
- 支持按指标名称、编码和取值方式查询指标。
- 指标编码在发布后自动生成，不允许在页面编辑。
- 指标名称字段展示为 `指标名称（考核事项）`。
- 支持人工评分和系统取值两种取值方式；新增默认人工评分。
- 系统取值指标需要为每条评分标准配置字段key，并配置是否允许人工修改。
- 支持指标草稿、发布启用、停用和删除；只有草稿指标允许删除。

## 字段与操作

| 类型 | 内容 |
| --- | --- |
| 查询条件 | 指标名称/编码、取值方式、左侧指标分类 |
| 列表字段 | 序号、指标编码、指标名称（考核事项）、指标分类、取值方式、允许人工修改、分数、状态、更新时间、说明 |
| 顶部操作 | 查询、重置、新建指标 |
| 行内操作 | 编辑、启用、停用、删除 |
| 编辑字段 | 指标名称（考核事项）、指标分类、取值方式、允许人工修改、说明、评分标准、字段key |

## 业务规则

- 启用中指标可被评分表配置选择；草稿中和已停用指标不进入评分表指标选择弹窗。
- 指标编码按 `METRIC-YYYYMMDD-序号` 在发布启用时生成；草稿指标不生成编码，列表提示“指标发布后生成”。
- 指标状态由系统动作自动维护：新建为草稿中，发布后为启用中，停用后为已停用；编辑表单不提供状态字段。
- 取值方式分为 `人工评分` 和 `系统取值`。
- 新增指标默认取值方式为人工评分，草稿或普通指标后续编辑可修改取值方式。
- 新增时选择系统取值，需要为每条评分标准输入字段key，并选择是否允许人工修改。
- 系统取值指标发布后，不允许在页面修改取值方式和编码，但允许调整是否允许人工修改。
- 默认一个指标有一条评分标准对应一个字段key；如果添加多个评分标准，每一条评分标准都需要独立字段key。
- 评分标准不配置关键项。
- 指标加入评分表后，评分表保留指标名称和评分标准快照，后续指标库口径变更不自动改写已配置评分表。
- 已启用或已停用指标不允许删除，避免影响历史评分表引用。
