## 当前需求

- 绩效确认不再拆分独立页面，由绩效管理页面根据数据权限、状态和当前处理人控制确认操作。
- 顶部不展示按角色区分的视图，左侧不展示说明文案。
- 状态视图按业务状态拆分为 `全部`、`待员工自评`、`待上级评价`、`待绩效确认`、`已确认/已归档`，卡片统计与当前可见数据一致；`已确认/已归档` 合并统计已确认和已归档数据。
- 列表支持按组织树、姓名/工号、考核周期、岗位、职级查询；列表人员范围为所有绩效薪资人员，未来将会补齐薪酬档案功能，页面列表上方不展示额外说明文字。
- 查询区提供 `待我处理` 开关，打开后只展示当前用户在当前状态下可处理的绩效记录，关闭后恢复普通列表；该筛选不再通过行背景色额外标记。
- 查询条件右侧提供 `批量确认绩效`，列表前置复选框；仅 `待绩效确认` 且当前用户为绩效确认人的记录可勾选和批量确认，未选择数据时需要提示先勾选绩效数据。
- `绩效表匹配规则` 属于工具/配置动作，放在查询区业务动作组靠后位置，不使用新增类主业务按钮样式。
- `绩效表匹配规则` 主承载改为抽屉；抽屉上方展示绩效规则列表，支持按优先级排序，抽屉下方通过标签页切换查看选中规则适用人员和异常人员。
- 配置规则列表固定行高，展示不全的字段单行省略，鼠标移入时通过浮层查看完整数据。
- 新增、编辑规则仍使用独立弹窗；规则字段包含规则名称、适用对象、适用职级、绩效表模板、评价流程、绩效确认人；适用对象可选择部门或人员。
- 绩效表匹配规则抽屉不出现横向滚动；适用对象和适用职级列表在表格中使用普通文本单行展示，超出省略，并通过鼠标移入浮层展示完整数据。
- 已修订不作为状态展示；绩效经过修改且待确认时，在姓名右侧展示 `改` 标识。
- 详情不按状态限制，所有可见数据都能通过 `详情` 打开。
- 绩效详情通过绩效管理页面或待办任务打开时都使用弹窗样式。
- 详情上方不展示绩效人员和流程信息概览，只保留绩效表内容和流程记录。
- 绩效详情中不展示评分表状态；草稿中的评分表不能在绩效业务中选择应用。
- 评分表通用填写组件在绩效详情中只展示基础信息和评分明细，不展示顶部统计卡片。
- `待绩效确认` 和 `已确认` 状态进入详情时底部展示 `保存修改`；若当前用户为该绩效记录的绩效确认人，则额外展示 `提交确认`。
- 评价流程只配置两级评价节点，绩效确认作为通用确认环节单独处理，不写入评价流程名称或节点。
- 列表展示 `绩效得分`，该分数为评分表按评分人权重计算出的最终得分，仅在员工自评和上级评价均完成后展示。
- `未匹配` 与 `不参与` 合并为同一状态 `无需考核`，表示根据绩效表配置未匹配绩效考核表，无需进入考核流程。
- 左侧组织树不展示标题；部门名前展示部门图标，姓名前展示人员性别图标，部门名右侧展示该部门及下级部门人员数量；人员节点只展示姓名，不附带员工岗位；组织树复选框默认不勾选。
- 操作列去掉修订、提交评价、退回自评和不参与等无实际意义入口；底部操作仅可能出现保存修改、提交/提交确认、退回、转派。
- 提交操作打开确认弹窗；退回操作打开原因填写弹窗；转派操作打开人员选择弹窗。
- 增加未匹配绩效表的部门节点和员工演示数据，用于展示无需考核逻辑。

## 业务规则

- 当前用户只看到本人、直属下属、当前处理人、绩效确认人或其管理部门范围内的绩效数据。
- `待我处理` 开关只保留当前用户可执行底部操作的记录，包括本人待自评、当前处理人为本人的待上级评价、绩效确认人为本人的待绩效确认；筛选结果不使用行背景色标记。
- 绩效表匹配规则按 `1 个规则名称 + N 个适用对象 + N 个适用职级 + 1 个绩效表模板 + 1 个两级评价流程 + 1 个绩效确认人` 保存为一条规则。
- 适用对象支持部门和人员混合多选；人员被多条规则匹配时仍按规则优先级取优先级较高的规则。
- `适用职级` 从当前职级字典中多选具体职级，例如 `P1`、`P2`、`M1`，不再选择 `P1-P3` 这类范围。
- 绩效表匹配规则抽屉上方按照规则优先级展示规则列表，排序操作与优先级合并展示；同一人员匹配多条规则时，适用优先级较高的规则。
- 点击 `新增规则` 或规则列表 `编辑` 后打开独立编辑弹窗，保存成功后关闭编辑弹窗并刷新抽屉内规则列表和命中人员列表。
- 抽屉下方通过标签页查看 `选中规则人员` 和红色 `异常人员`，字段包含序号、姓名、区域、部门、岗位-职级；异常人员指有绩效薪资但未匹配到任何绩效规则的人员。
- 评价流程下拉选项只维护两级评价流程，例如 `自评+直属上级评`、`运营中心评价流程`；绩效确认人是通用确认配置，不属于评价流程节点。
- `绩效得分` 来源于绩效表评分结果，按评分表配置的评分人权重计算最终分；待员工自评和待上级评价阶段不展示得分。
- 待员工自评：员工本人在详情底部看到 `提交`，提交后流转至直属上级评价。
- 待上级评价：当前处理人在详情底部看到 `提交`、`退回`、`转派`。
- 待绩效确认：详情底部展示 `保存修改`；绩效确认人额外看到 `提交确认`，提交后状态变更为 `已确认`，列表同步更新状态。
- 已确认：详情底部展示 `保存修改`，用于补录或修正已确认绩效内容；点击后提示保存会退回 `待绩效确认`，确认保存后列表状态更新为 `待绩效确认`，当前处理人更新为绩效确认人。
- 批量确认只处理已勾选且绩效确认人为当前用户的数据；未勾选数据时提示先选择绩效数据，已勾选但无可确认数据时提示用户选择正确数据。
- 已确认后当前处理人清空，列表为空白展示，修订标识清除。
- 绩效是本月结束后评价本月，归档日期暂定为次月 5 日；同一考核周期的绩效文件在固定日期统一变更为 `已归档`，不会单条单独归档。列表不展示归档日期，仅在业务规则中保留该归档节奏。
- `改` 标识只在 `modified = true` 且状态为 `待绩效确认` 时展示，位置在姓名右侧。
- `无需考核` 表示员工所属部门或职级未匹配绩效表配置，不再单独展示 `未匹配` 或 `不参与` 状态。
- 左侧组织树默认不勾选任何范围；未勾选时列表展示当前用户权限范围内的全部数据。
- 退回自评后状态变为 `待员工自评`，当前处理人变为员工本人。
- 转派后只调整当前处理人，不改变绩效确认人和业务状态。
- 提交自评、提交评价、提交确认、批量确认、退回自评、保存修改和转派完成后，列表需要同步更新对应状态或当前处理人，并通过完成提示告知用户更新结果。
- 从绩效详情底部发起的保存修改、提交、提交确认、退回和转派在执行成功后关闭详情弹窗；保存修改、提交和提交确认使用页面内受控确认弹窗承载，确保从详情弹窗内点击时可稳定打开。
- 详情弹窗始终可打开；无可执行权限时底部不展示操作按钮。
- 待办任务进入绩效处理时打开绩效管理目标记录弹窗，不提供独立绩效确认页。

## 待确认问题

- 批量确认是否需要二次校验每条绩效表的必填评分项，当前原型仅按状态和权限校验。
- 已确认绩效是否允许后续发起修订，当前原型只展示详情，不开放修订入口。
- 无需考核人员是否需要单独的异常处理入口，当前仅通过隐藏开关控制展示。
