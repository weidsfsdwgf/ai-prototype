# 评分表配置

## 基本信息

| 项 | 内容 |
| --- | --- |
| 所属系统 | OA |
| 一级目录 | OA-绩效管理 |
| 二级目录 | 评分表配置 |
| 菜单路由 | `/oa/performance/scorecards` |
| 文档状态 | 持续迭代 |
| 最近更新 | 2026-05-04 |

## 当前需求

- 支持统一维护多类考核评分表模板。
- 支持配置基础字段、自定义字段、评分人、考核维度、考核事项、评分标准和分值。
- 支持从指标库选择已启用指标，作为当前考核维度下的考核事项加入评分表。
- 支持草稿中、启用中、已停用三类模板状态。
- 支持模板发布前校验和评分表预览。

## 字段与操作

| 类型 | 内容 |
| --- | --- |
| 列表字段 | 模板编码、评分表名称、状态、负责人、字段数、评分人、总分、版本、更新时间、说明 |
| 顶部操作 | 新建评分表、查询、重置 |
| 行内操作 | 编辑、预览、复制、发布、停用、删除 |
| 编辑操作 | 保存配置、预览填写窗口、从指标库选择；编辑表单不维护状态、编码和版本 |

## 业务规则

- 新建模板默认为草稿中。
- 模板状态由系统动作自动维护：新建为草稿中，发布后为启用中，停用后为已停用；编辑表单不提供状态字段。
- 评分表配置页不绑定具体业务类型，各业务按自身场景选择已发布模板编码发起评分。
- 评分表编码由系统统一生成，生成时机为模板发布；草稿状态不生成编码，列表中以灰色“模板发布后生成”提示。
- 评分表编码规则为 `SCORE-YYYYMMDD-序号`，例如 `SCORE-20260505-001`。
- 每次编辑保存自动更新模板版本号，编辑表单不允许手工维护版本。
- 通过阈值、优秀阈值和结果名称不在通用模板中配置，由使用评分表的业务自行定义。
- 一张评分表至少保留一个评分人，最多支持五个评分人。
- 每张评分表至少包含一个考核维度，每个维度至少包含一个考核事项，每个事项至少包含一个评分标准。
- 编辑评分表配置的考核内容区域采用可折叠维度卡片；考核维度、考核事项、评分标准和分值分别通过表单标签和层级缩进区分，维度右侧操作按钮与输入框对齐。
- 从指标库选择指标时，只允许选择启用中指标；已在当前评分表中引用的指标不可重复选择。
- 指标库指标仅作为考核事项加入评分表；考核维度仍由评分表本身新增和维护。
- 从指标库选择指标时，需要先在评分表中定位考核维度，再把所选指标加入该维度下。
- 指标加入评分表后，指标名称作为考核事项，指标评分标准同步为评分表评分标准。
- 从指标库选择加入的考核事项不允许在评分表中修改名称、评分标准描述、分值或增删评分标准；如选错指标，可删除该考核事项后重新选择。
- 分值只配置在评分标准层级，分值必须为非负数字。
- 模板基础信息、字段、评分人和考核内容完整后才允许发布启用。
- 已停用模板不再用于新建评分，但仍保留查询和归档。
- 删除模板需二次确认，且仅未发布的草稿模板允许删除。
- 系统字段支持搜索、勾选和排序，以适配字段数量较多的业务场景；系统字段均由系统值自动带出，不区分必填和选填。
- 负责人从用户列表中下拉选择。
- 各业务模块通过通用评分填写窗口打开已配置模板，窗口按模板动态渲染基础字段、评分人、考核维度、考核事项和评分标准。
- 评分填写窗口中的系统字段自动带出且不可编辑，预览标签仅展示字段名；自定义字段可填写。
- 评分明细按每条评分标准逐行填写，考核维度、考核事项采用类似 Excel 合并单元格的汇总展示。
- 多个评分人不分块展示，评分列按评分人数量横向展开，表头按“评分人（权重xx%）”命名；未配置权重时展示“评分人（等权重）”。
- 配置人可在编辑抽屉中直接预览填写窗口，也可在列表中使用“预览”验证业务评分时的实际填写效果。

## 原型实现记录

- 已新增 `/oa/performance/scorecards` 页面。
- 已接入 OA 二级目录“OA-绩效管理”和菜单“评分表配置”。
- 旧路由 `/oa/config/scorecards` 保留跳转到 `/oa/performance/scorecards`。
- 已实现模板列表、筛选、编辑抽屉、系统字段选择排序、动态评分项、发布校验、填写预览和删除确认。
- 已在评分表编辑抽屉的考核维度内新增指标库选择弹窗，支持按关键字和指标分类筛选指标；弹窗列表展示指标说明，不展示字段key。
- 已抽出通用 `ScorecardScoringDrawer` 组件，供转正、课程、项目等业务以模板为参数发起评分填写。
- 当前数据为前端静态样例，覆盖转正评估、课程质量评估、阶段性考核评估和项目复盘评估。
