## 当前需求

- 页面采用常见组织维护布局：左侧部门树，右侧人员列表。
- 左侧部门树展示组织层级和当前部门摘要。
- 支持修改部门信息。
- 右侧人员列表遵循标准列表框架，支持查询人员并修改人员所在部门。

## 页面结构

| 区域 | 内容 |
| --- | --- |
| 左侧部门树 | 部门搜索、部门树、当前部门编码、上级部门、负责人、成员数 |
| 右侧筛选区 | 姓名/员工编号、部门、状态、查询、重置 |
| 右侧列表 | 部门下人员清单 |
| 弹窗 | 修改部门信息、调整人员所在部门 |

## 人员列表字段

## 功能操作

| 类型 | 操作 |
| --- | --- |
| 部门操作 | 修改部门信息 |
| 人员操作 | 调整部门 |

## 业务规则

- 修改部门信息包括部门名称、上级部门、负责人、状态。
- 部门编码系统生成，不能直接编辑。
- 调整人员所在部门时，需要选择调整后部门和主部门。
- 人员列表行内操作使用 `TableActions`，当前仅展示“调整部门”。
