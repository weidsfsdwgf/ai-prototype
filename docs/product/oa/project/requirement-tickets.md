# 需求工单

- 所属系统：项目
- 一级目录：项目
- 二级目录：需求工单

- 最近更新：2026-05-15

## 当前需求

根据《需求工单》流程建设可演示的原型页面，支持按分类维护业务负责人、评审人、IT 业务负责人；业务人员新增需求工单、暂存或提交审批；审批通过后由 IT 负责人确认、转派并关联团队管理中的 IT 需求。工单状态根据审批阶段和关联 IT 需求进展展示。

## 字段与操作

左侧分类树：

- 根节点名称为“所有”，分类树用于筛选需求工单。
- 树上方提供模块搜索。
- “所有”右侧提供新增一级模块入口。
- 其他模块右侧 Popover 提供编辑模块、添加子模块、删除模块操作。
- 分类信息中维护业务负责人、评审人、IT 业务负责人。
- 新增工单时默认取当前分类对应负责人，不在弹窗中重复展示。

列表查询字段：

- 需求标题 / 背景
- 工单号
- 提出人
- 状态

列表展示字段：

- 序号、工单号、需求标题、所属模块、需求类型、标签、优先级、提出人、IT 负责人、状态、关联需求数、期望上线日期、更新时间。

主操作：

- 新增需求工单：打开弹窗填写需求标题、需求模块、需求类型、优先级、标签、期望上线日期、提出人、背景、痛点、需求描述、附件和价值类补充信息。
- 提出人为下拉选择，默认当前用户。
- 需求背景、业务痛点默认单行高度，可随输入自动展开。
- 仅保存：工单进入待提交状态。
- 提交审批：工单进入审批中状态。

行操作：

- 点击数据行：抽屉展示简约步骤条和两个标签页。
- 详情说明标签页：展示基础信息、富文本需求描述、关联需求列表和评论输入框。
- 流程记录标签页：去掉审批链路，统一展示创建、审批、关联需求、评论、更新等操作留痕。
- 转派负责人：IT 负责人为当前用户时，列表负责人旁展示快捷图标打开修改负责人弹窗。
- IT 负责人和关联需求旁的小图标使用带背景的发送按钮。
- 关联需求：展示格式为 `已完成数量/总需求数量`，例如 `1/2`，左对齐展示；当前用户为 IT 负责人时，数字旁展示发送按钮，点击打开关联需求列表，并提供关联创建需求、关联已有需求按钮。
- 关联需求弹窗字段：需求编号、需求名称、状态、负责/验收、开发人员；操作仅保留取消关联。
- 关联创建需求弹窗字段：需求名称、团队、分类、提出人、负责/验收、计划完成日期、优先级、标签、描述。
- 关联已有需求弹窗：上方查询条件，下方需求列表，选中需求后点击批量关联。
- 编辑：用户为提出人或创建人，且状态为待提交、已驳回、已撤回时展示。
- 提交审批：用户为提出人或创建人，且状态为待提交、已驳回、已撤回时展示。
- 删除：用户为提出人或创建人，且状态为待提交、已驳回、已撤回时展示。
- 审批详情：状态为审批中时展示。
- 撤回：用户为提出人或创建人，且状态为审批中时展示。
- 关闭工单：用户为 IT 负责人，且状态为待确认、待排期、开发中时展示。

## 业务规则

- 只有被添加到模块下的业务人员，才能在对应模块中提交需求。本原型以需求模块选择模拟该权限前置。
- 审批链路分两级：业务负责人审核、评审人评估审核。
- 审批通过后工单进入待确认，由分类下配置的 IT 业务负责人默认承接。
- 一个需求工单可以关联多个 IT 需求。
- 关联 IT 需求后，工单状态根据关联需求状态同步。
- 工单待确认：无关联需求，或关联需求全部为待开发。
- 工单开发中：关联需求存在开发中、待测试、测试通过、待发布、待验收任一状态。
- 工单已完成：关联需求全部为已完成。
- IT 内部需求按迭代排期，迭代需要先确认再开始；如果在进行中迭代加入需求，需求标记为插队；如果未完成需求从已确认迭代移出，则标记为流转。
- 需求类型只有普通需求、价值需求。
- 价值需求需要补充验收标准或成功标准、核心能力跃升、预计价值量化。
- 核心能力跃升支持多选：管理赋能、决策转型、能力复制。
- 预计价值量化支持营收增长、效率提升、成本降低，必须至少选择一项；选中后补充对应数值。
