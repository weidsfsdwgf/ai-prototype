## 当前需求

- 页面用于汇总标准作业流程中的需求工单、团队迭代进度、人员任务进度与 Bug 处理概览。
- 页面上方展示四个数字统计卡片：本周新增工单数、待确认工单、开发中工单数、本周上线需求数。
- 页面主体采用 2×2 四卡片布局：工单概览、迭代进度、任务进度、Bug概览。
- 主体四个卡片保持一致宽高。
- 工单概览支持按模块分类、IT负责人、提出人三个维度切换。
- 迭代进度按团队维度展示，迭代筛选紧跟在“迭代进度”标题右侧。
- 任务进度按人员维度展示，与迭代进度共用当前选择的迭代范围。
- Bug概览按团队维度展示 Bug 总数、待处理、已处理、待关闭。

## 字段与操作

| 区域 | 字段或操作 |
| --- | --- |
| 页面操作 | 刷新、导出 |
| 数字统计 | 本周新增工单数、待确认工单、开发中工单数、本周上线需求数 |
| 工单概览视图 | 模块分类、IT负责人、提出人 |
| 工单概览字段 | 完成进度（已完成/工单总数）、需求工单总数、待确认、开发中、待验收、已完成、实际工时 |
| 迭代进度 | 团队、进度、需求总数、未完成、已完成、预计工时、实际工时、剩余工时 |
| 任务进度 | 部门、人员、进度、任务总数、未完成、已完成、预计工时、实际工时、剩余工时 |
| Bug概览 | 团队、bug总数、待处理、已处理、待关闭 |
| 表头交互 | 文本维度字段支持筛选，数量、进度和工时字段支持排序 |

## 业务规则

- 完成进度 = 已完成需求工单数量 / 需求工单总数。
- 完成进度字段后展示“已完成/工单总数”，不展示百分比。
- 工单概览中的进度条总长按当前维度下的工单总数缩放；不同总数的行会呈现不同长度。
- 需求工单总数前置问号图标，鼠标移入问号图标时展示“仅包含审批通过后且未被关闭的工单”。
- 迭代进度以团队为维度聚合迭代周期覆盖当前日期的团队迭代。
- 任务进度以任务负责人为维度聚合，并展示所属部门。
- Bug概览以团队为维度聚合当前迭代范围内的 Bug 处理状态。
