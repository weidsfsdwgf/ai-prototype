## 当前需求

- 页面定位调整为统一的 OA 申请发起入口，转正申请只是其中一个申请类型。
- 页面分为两个标签页：`发起申请` 和 `数据记录`。
- `发起申请` 按申请分类展示申请入口，入口卡片仅保留申请名称和图标，点击后打开表单抽屉，填写数据并提交审批。
- `发起申请` 不强制套用标准列表页规范，采用现代、简洁的入口型布局。
- `数据记录` 直接展示管理员视图，不再提供普通用户/管理员视角切换。
- `数据记录` 参考标准框架页，左侧展示一级分类和二级申请类型；一级分类仅负责展开折叠，点击二级申请类型后右侧才展示对应列表。

## 框架页映射

| 功能区 | 采用框架 | 设计说明 |
| --- | --- | --- |
| 发起申请 | 入口型工作台 | 按人事、行政、财务、采购、合同分类展示小型申请入口 |
| 申请表单 | 标准抽屉表单 | 根据申请类型动态渲染字段，提交后生成审批中记录 |
| 数据记录分类 | 标准左侧分类 | 分类包含一级业务分类、二级申请类型，不做数据汇总 |
| 数据记录列表 | 标准列表页 | 查询区、权限切换、表格、分页、`TableActions` 操作列 |
| 申请详情 | 审批详情抽屉 | 展示审批信息、申请摘要和审批链路，支持全屏 |
| 撤销申请 | 标准弹窗 | 管理员视图下进行中的申请支持撤销 |

## 字段与操作

| 类型 | 内容 |
| --- | --- |
| 标签页 | 发起申请、数据记录 |
| 申请分类 | 人事、行政、财务、采购、合同；一级分类下展示转正申请、请假申请、出差申请、费用报销、采购申请、用印申请等二级节点 |
| 申请类型 | 转正申请、请假申请、出差申请、费用报销、采购申请、用印申请 |
| 表单字段 | 由申请类型配置动态生成，支持输入框、多行文本、下拉、日期区间、数字、上传 |
| 转正申请查询条件 | 姓名/员工编号/手机号、区域、部门、岗位、负责HR、状态、入职日期起止、实际转正日期起止 |
| 转正申请列表字段 | 序号、员工编号、姓名、区域、部门、岗位-职级、入职日期、预计转正日期、实际转正日期、状态、负责HR、提交时间、更新时间、操作 |
| 通用申请列表字段 | 序号、审批单号、流程名称、分类、发起人、部门、摘要、状态、结果、当前节点、发起时间、结束时间、操作 |
| 行内操作 | 详情、撤销 |
| 列表操作 | 导出 |

## 业务规则

- 申请类型状态为 `启用` 时可发起；状态为 `停用` 时入口保留但不可发起。
- 发起申请提交后生成一条 `进行中 / 处理中` 的 OA 申请记录，并自动切换到 `数据记录` 标签页。
- `数据记录` 默认没有选中具体表单时，右侧展示空状态提示。
- 一级分类节点只负责展开和折叠，不触发右侧列表查询。
- 二级申请类型节点按流程名称过滤右侧列表，例如点击 `转正申请` 展示转正申请列表。
- 左侧分类不展示数据汇总数量。
- 申请分类树采用接近系统菜单的浅色样式，保持简约、轻量。
- 转正申请列表中的实际转正日期直接展示日期，不追加 `待回写` 标记。
- 进行中的申请支持撤销；通用申请撤销后状态更新为 `已结束`，结果更新为 `已撤回`；转正申请撤销后状态更新为 `已撤销`。
- 转正申请使用转正申请详情抽屉展示员工信息和申请文件；其他申请使用通用审批详情抽屉。
