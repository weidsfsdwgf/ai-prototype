# 薪酬统计功能交付说明

## 文档目的

本文档用于向外部团队、第三方项目或非本项目开发人员交付“薪酬统计”功能说明。文档只描述业务目标、功能范围、数据契约、交互规则、计算口径和验收方法，不依赖任何特定代码仓库、前端框架、组件库或文件路径。

阅读者应能够基于本文档，在任意合适的技术栈中重新设计和实现薪酬统计功能，并通过验收清单验证实现是否完整。

## 功能概述

薪酬统计用于查询、归档和复核每个薪酬周期已经审批完成的薪酬数据。功能面向薪酬、财务、人力资源、部门管理者等角色，帮助他们从组织、部门、员工三个层级查看薪酬结果、考勤影响、绩效提成、补贴奖金和扣款调整。

该功能的核心不是发起或办理薪酬审批，而是在审批完成后形成稳定的历史快照，支持按月份查看、按组织或部门筛选、查看员工明细、核对汇总口径，并保留可追溯的归档信息。

## 使用角色

| 角色 | 使用诉求 |
| --- | --- |
| 薪酬专员 | 查询某月薪酬归档结果，核对员工明细、部门汇总和异常调整。 |
| 人力资源负责人 | 查看组织薪酬分布、部门人数、实发合计、人均薪酬和考勤影响。 |
| 财务人员 | 核对实发工资、代扣扣款、个税社保、公积金等发放相关数据。 |
| 部门负责人 | 查看本部门及下级部门员工薪酬结果和绩效提成明细。 |
| 审计或管理人员 | 追溯历史月份薪酬快照，确认数据来自审批通过后的归档版本。 |

## 功能范围

### 必须包含

- 按薪酬月份查询已归档薪酬数据。
- 查看月度归档记录的基础信息，包括月份、组织、提交人、提交时间、归档状态。
- 查看部门维度统计，包括算薪人数、应发合计、实发合计、人均薪酬、考勤、绩效、提成、奖金和补贴。
- 查看员工维度明细，包括基础信息、薪资结构、考勤假勤、绩效提成、补贴奖金、扣款调整、应付工资和实付工资。
- 支持部门树筛选、员工筛选、表格字段筛选和重置筛选。
- 支持查看员工薪资详情。
- 支持查看部门指标的构成明细。
- 支持以审批完成时的数据快照作为统计来源，避免历史数据被组织或员工资料的后续变化污染。
- 提供可验证的计算口径和验收清单。

### 不包含

- 不负责发起薪酬审批。
- 不负责审批通过、驳回、撤回、加签、转交等流程动作。
- 不负责工资实际发放、银行代发或财务付款。
- 不强制要求提供跨月趋势、图表分析或预测模型。
- 不强制要求提供导出、打印、权限配置和操作审计；这些可作为扩展能力。

## 页面结构

建议页面由以下区域组成：

| 区域 | 说明 |
| --- | --- |
| 顶部筛选区 | 选择薪酬月份、组织或薪酬对象类型，可扩展审批单号、归档状态、发起人等条件。 |
| 视图切换区 | 在“员工视图”和“部门视图”之间切换。 |
| 操作区 | 提供重置筛选、列配置、导出等操作；导出为可选能力。 |
| 左侧组织树 | 展示组织、部门和员工层级，用于限定统计范围。 |
| 右侧数据表 | 根据当前视图展示员工薪酬明细或部门统计结果。 |
| 明细弹窗 | 展示员工薪资详情或部门指标构成明细。 |

## 数据模型

### 月度归档记录

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `archiveId` | string | 是 | 归档记录唯一标识。 |
| `payrollMonth` | string | 是 | 薪酬月份，格式建议为 `YYYY-MM`。 |
| `organizationId` | string | 是 | 组织标识。 |
| `organizationName` | string | 是 | 组织名称。 |
| `approvalNo` | string | 否 | 对应薪酬审批单号。 |
| `submittedBy` | string | 是 | 发起人、提交人或归档人。 |
| `submittedAt` | datetime | 是 | 审批提交或归档生成时间。 |
| `approvedAt` | datetime | 否 | 审批通过时间。 |
| `archivedAt` | datetime | 否 | 归档完成时间。 |
| `archiveStatus` | enum | 是 | 建议值：`archiving`、`archived`、`failed`。 |
| `version` | string | 否 | 归档版本号；如支持修订，必须提供。 |
| `departmentSnapshot` | array | 是 | 部门树快照。 |
| `employeeSnapshot` | array | 是 | 员工薪酬快照。 |

### 部门快照

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `departmentId` | string | 是 | 部门唯一标识。 |
| `parentDepartmentId` | string | 否 | 上级部门标识，根节点为空。 |
| `departmentName` | string | 是 | 部门名称。 |
| `managerName` | string | 否 | 部门负责人。 |
| `level` | number | 是 | 部门层级，从 1 开始。 |
| `children` | array | 否 | 下级部门。 |

部门统计字段可以存储为快照，也可以在前端或后端基于员工快照实时计算。无论采用哪种方式，最终展示口径必须与“计算口径”章节一致。

### 员工薪酬快照

| 类别 | 字段 |
| --- | --- |
| 基础信息 | 员工 ID、姓名、性别、区域、所属部门、岗位、职级、入职日期、转正日期、预计转正日期、离职日期 |
| 薪资结构 | 薪资合计、薪资结构说明、应付工资、上月薪资差额、审核调整、备注 |
| 考勤假勤 | 应出勤、实出勤、请假天数、请假明细、迟到次数、补卡次数、加班时长、加班明细 |
| 绩效提成 | 绩效工资、绩效分数、实发绩效、实发提成、剩余提成、提成明细 |
| 补贴奖金 | 综合补贴、假勤补贴、业务补贴、福利补贴、奖金及其明细 |
| 扣款调整 | 考勤扣款、其他扣款、代扣扣款、代扣明细 |
| 结算结果 | 应付工资、实付工资、调整说明 |

## 关键业务规则

- 薪酬统计只展示已完成审批并进入归档流程的数据。
- 每个月的统计结果必须来自该月审批通过时的快照。
- 快照应包含当时的组织结构、部门归属、员工基础信息和薪酬结算明细。
- 员工后续调岗、离职、改名、职级变化，不应影响历史月份归档结果。
- 如允许归档修订，必须保存版本号、修订原因、修订人、修订时间，并能区分当前版本和历史版本。
- 归档统计页默认不应直接修改正式薪酬结果；如果保留备注或调整能力，必须明确该能力是否会回写归档数据。
- 部门维度统计应覆盖所选部门及全部下级部门员工。
- 筛选后的表格汇总应基于当前可见数据重新计算。

## 交互规则

### 月份与组织筛选

- 默认展示最近一个已归档薪酬月份。
- 月份不可为空；如果用户清空月份，应自动恢复为默认月份或提示必须选择月份。
- 选择不存在归档数据的月份时，应显示空状态，并提示该月份暂无归档数据。
- 如存在多个组织或薪酬对象类型，应支持切换；不可用类型应展示禁用状态或不展示。

### 员工视图

- 默认展示员工维度列表。
- 左侧组织树支持按部门或员工姓名搜索。
- 勾选部门后，右侧只展示该部门及下级部门员工。
- 勾选员工后，右侧只展示被选员工。
- 员工姓名可点击进入薪资详情。
- 表格应支持字段筛选、横向滚动和固定关键列。
- 表格底部建议展示合计和均值。

### 部门视图

- 部门视图应支持树状列表。
- 可选支持平铺列表，用于跨层级筛选和比较部门。
- 点击部门名称可下钻到该部门员工视图。
- 部门指标中的明细类金额可点击查看构成，例如提成明细、绩效明细、考勤扣款明细、奖金明细、福利补贴明细。

### 重置筛选

重置筛选应恢复以下状态：

- 清空部门或员工选择。
- 清空表格列筛选。
- 清空搜索关键字。
- 清空明细分类筛选。
- 保留当前薪酬月份和组织选择。

## 计算口径

### 员工实付工资

```text
实付工资 = 应付工资 + 审核调整
```

如果系统不允许在统计页调整数据，则审核调整直接取归档快照值。如果允许临时调整预览，则实付工资展示应优先使用当前页面的临时调整值，并明确该值是否保存。

### 部门统计

| 指标 | 计算方式 |
| --- | --- |
| 算薪人数 | 当前部门及下级部门员工数量。 |
| 应发工资合计 | 员工应付工资求和。 |
| 实发工资合计 | 员工实付工资求和。 |
| 人均薪酬 | 实发工资合计 / 算薪人数；人数为 0 时显示 0 或 `-`。 |
| 提成合计 | 员工实发提成求和。 |
| 人均提成 | 提成合计 / 算薪人数；人数为 0 时显示 0 或 `-`。 |
| 绩效合计 | 员工实发绩效求和。 |
| 人均绩效 | 绩效合计 / 算薪人数；人数为 0 时显示 0 或 `-`。 |
| 平均绩效分 | 员工绩效分数求和 / 算薪人数；人数为 0 时显示 0 或 `-`。 |
| 奖金合计 | 员工奖金求和。 |
| 福利补贴合计 | 员工福利补贴求和。 |
| 出勤率 | 实出勤合计 / 应出勤合计 * 100%；应出勤为 0 时显示 0 或 `-`。 |
| 人均请假 | 请假天数合计 / 算薪人数。 |
| 人均迟到 | 迟到次数合计 / 算薪人数。 |
| 人均补卡 | 补卡次数合计 / 算薪人数。 |
| 人均加班 | 加班时长合计 / 算薪人数。 |
| 人均考勤扣款 | 考勤扣款合计 / 算薪人数。 |

### 表格汇总

- 合计行：对当前筛选结果中的数值字段求和。
- 均值行：对当前筛选结果中的数值字段求平均。
- 非数值字段不参与汇总。
- 金额建议使用千分位展示，保留 0 到 2 位小数，具体精度由业务确定。

## 验收清单

### 功能验收

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| A1 | 进入薪酬统计页面 | 能看到月份筛选、视图切换和薪酬数据区域。 |
| A2 | 默认月份 | 默认展示最近已归档月份。 |
| A3 | 月份切换 | 切换月份后，部门、员工、汇总和明细均切换到对应快照。 |
| A4 | 无数据月份 | 展示清晰空状态，不报错。 |
| A5 | 员工视图 | 能展示员工基础信息、薪酬、考勤、绩效、提成、补贴、扣款和结算结果。 |
| A6 | 部门筛选 | 选择部门后，仅展示该部门及下级部门员工。 |
| A7 | 员工详情 | 点击员工可打开薪资详情，字段完整且与列表一致。 |
| A8 | 部门视图 | 能展示部门层级和部门统计指标。 |
| A9 | 部门下钻 | 点击部门后能查看该部门员工。 |
| A10 | 指标明细 | 点击可钻取指标时，能看到构成明细。 |
| A11 | 重置筛选 | 重置后筛选条件恢复默认，但保留当前月份。 |
| A12 | 历史快照 | 员工或部门主数据变更后，历史归档数据不被覆盖。 |

### 数据验收

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| D1 | 员工实付工资 | 等于应付工资加审核调整。 |
| D2 | 部门算薪人数 | 等于部门及下级部门员工数量。 |
| D3 | 部门实发合计 | 等于部门范围内员工实付工资求和。 |
| D4 | 部门人均薪酬 | 等于部门实发合计除以算薪人数。 |
| D5 | 出勤率 | 等于实出勤合计除以应出勤合计。 |
| D6 | 表格合计 | 等于当前筛选结果的字段求和。 |
| D7 | 表格均值 | 等于当前筛选结果的字段平均值。 |
| D8 | 明细一致性 | 员工详情、部门明细和列表展示的关键金额一致。 |

### 非功能验收

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| N1 | 可读性 | 表格字段名称清晰，金额、数字、百分比格式统一。 |
| N2 | 可追溯 | 能识别数据所属月份、组织、归档时间和版本。 |
| N3 | 稳定性 | 大量员工和多级部门下，筛选、切换和弹窗不出现明显卡顿或崩溃。 |
| N4 | 权限 | 无权限用户不可查看敏感薪酬数据；具体权限模型由接入项目定义。 |
| N5 | 安全 | 薪酬数据传输、存储、导出应符合接入项目的数据安全要求。 |

## 推荐交付物

对外交付该功能时，建议至少提供：

- 功能说明文档。
- 数据字段字典。
- 计算口径说明。
- 原型图或页面截图。
- 验收用例。
- 至少一份脱敏样例数据。
- 权限与数据安全说明。

## 扩展能力

后续可按业务成熟度增加：

- 月度归档列表。
- 多版本归档和差异对比。
- 跨月趋势分析。
- 按组织、部门、岗位、职级统计。
- 导出 Excel 或 PDF。
- 异常薪酬预警。
- 审计日志。
- 与财务付款、银行代发或工资条系统对接。
